# 提交错误的解决方法
## 一、版本回滚（撤销错误提交）
1. 错误提交还在本地（**未推送到远程**）  
    * 用git reset撤销提交：
git reset --soft HEAD^(撤销最后一次提交，保留修改内容)  
git reset --hard HEAD^（撤销最后一次提交，同时丢弃修改内容）  
2.  错误提交**已经推送到远程**（先在本地回滚，再强制推送到远程）  
   * git reset --hard HEAD^(本地回滚到上一个版本)  
  git push -f origin 分支名  
## 二、冲突解决
**当多人修改同一文件的同一部分时，Git 会提示冲突，需要手动解决**
1. 处理文件的冲突地方  
    1. 输入vim "文件名.md"  
    2. 找到冲突区域标志（<<<<<<<和>>>>>>>和=======）  
    3. 按i进入插入模式，可进行编辑 
    4. 编辑完成后按ESC键退出插入模式，输入:wq保存并退出 
2. 将处理完的文件加入暂存区  
3. 最后提交到仓库

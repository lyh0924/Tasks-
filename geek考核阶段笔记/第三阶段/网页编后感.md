## 项目构想
我在接收到考核项目大纲的时候，就把四个阶段的任务都大致浏览了一遍，看到第三个阶段的任务是制作一个网页时，就已经开始思考我要做一个什么样的网页了，有想过制作一个有关我喜欢的卡通形象的网页，也有想过制作一个个人介绍的网页……

但让我最终确定我的网页内容的是我无意中刷到的一个宣传家乡的视频，我就想到我可以制作一个网页来介绍我的家乡，我认为这是一个很有意义的事情，刚开始我在我脑中设想了一个非常精美的网页，但我发现我所学的知识并不能完全展示我的设想，但我还是在有限的时间内，尽可能多的学习HTML，css，js这些曾经完全陌生的事物。

最终还是做出来了一个算是完整的网页，即使设想中的很多东西，我现阶段还做不出来，网页也没有自己想象中的那样精美，但我还是很欣慰自己能够做出来这个网页。
## 利用HTML搭建框架
我这个网页的制作，主要包含了**四大部分：顶部导航 全屏视频  右侧导航 景点板块**
1. **顶部导航**：`<div class="top-nav">Welcome to Beihai</div>`
2. **全屏视频**：
```
<div class="hero">
  <video src="..."></video>
  <div class="right-nav">...</div>
</div>
```
在视频的这个盒子里面，还放了右侧的导航
3. **右侧导航**：
在这个网页中我认为这个利用锚点链接的跳转，通过点击景点的名称就可以跳转到对应景点的介绍处，是一个比较有意思也是很常在网页或网站中见到的设计，也算我是活学活用的一处。
`<a href="#yintan">银滩</a>`等
4. **景点板块**：
这个景点板块是整个html的核心，它包含了景点的标题以及对于六个景点的介绍，还有景点的图片背景。
```
<div id="yintan">
  <h2>北海银滩</h2>
  <p>介绍...</p>
</div>
```
## 利用CSS美化页面
1. **去掉浏览器默认间距，让布局按照自己的想法和设计来**
```
* {
  margin:0; padding:0; box-sizing:border-box;
}
```
2. **在锚点链接发挥作用的时候，让页面平滑滚动，显得更柔和一些**
`html { scroll-behavior: smooth; }`
3. **使顶部导航始终处于最上端，让浏览者能明晰浏览内容和主题**
```
.top-nav {
  position: fixed;
  top:0;
  width:100%;
  background:#61aff4;
  z-index:999;
}
```
4. **首屏视频全屏**:
让打开页面的浏览者第一眼就能沉浸在北海的景色中，提高愿意了解的兴趣度
```
.hero { height:100vh; }
video { object-fit:cover; }
```
5. **右侧固定悬浮导航**：
这个导航也是在听了网上视频里涉及到的fix定位知识点后，认为可以加入的，只不过我这个页面并不是很长，不能完全凸显它的作用，但是还是为浏览者提供了便捷，这个导航会一直居中贴在右边，就算滚动页面也不动。
```
.right-nav {
  position:fixed;
  right:30px;
  top:50%;
  transform:translateY(-50%);
}
```
6. **按钮样式（圆角、半透明、 hover 放大）**
这些样式也是在进行了网上的视频学习后学到的，学以致用一下
```
.nav-btn {
  border-radius:20px;
  background:rgba(255,255,255,0.2)
}
.nav-btn:hover { transform:scale(1.1); }
```
7. **全屏的景点背景覆盖**
```
.spot-section {
  min-height:100vh;
  background-size:cover;
  background-position:center;
}
```
8. **半透明黑色背景（让文字更清晰）**
```
.spot-section::before {
  background:rgba(0,0,0,0.4);
}
```
9. **文字样式（白色、阴影、居中）**
```
.spot-content {
  color:white;
  text-shadow: ...;
}
```

## 困难及解决  
我在这个网页制作的过程中遇到了不少的困难😔

首先是因为一开始进行视频学习，学的都是一些零散的知识点，如果告诉我让我单独要把哪个知识点涉及的代码写出来还能写，但是如果说在一个完整的网页中要熟练的运用所学的知识点，要在哪里用到哪个知识点，并且要把html和css衔接起来成为一个整体，成了一个难题，我觉得这个是我遇到的第一个困难，就是感觉脑海里面已经构想出了一个网页，但是一开始要打字的时候就不知道该怎么写了

之后我认为之所以会有这样的烦恼，首先是我的思路框架不够清晰，其次的话就是我对于那些知识点的掌握的熟练程度还不够
因此我就先用一张纸把我所设想的HTML框架大概画了下来，再看一下哪些地方可以用到学到的css的美化，然后再在框架上添加进去，等做完了这些之后，我再对于知识点进行了整体复习

最后开始着手写网页的时候就没有那么手足无措了，感觉这是一个把一个整块拆解成一个个小部分，再进行合并的过程，虽然过程中有很多困难，但最后看到成果还是很开心的

在制作完整个网页的最后，因为要生成一个能让大家点进去都能看到的链接，这也是我遇到的最后一个需要解决的困难

我选择了在GitHub中的Pages里面生成链接，在生成链接的过程中都挺顺利的，但是在点开生成的链接之后我就看到，生成的网页是我之前做过的一个已经淘汰的一个版本，但是不知道为什么这个链接打开后显示的还是旧的，并且在我检查之后，我发现我的代码也是最新的，经过AI分析，有可能是因为缓存问题

最后我就进行了网页的强制刷新这时候底下的图片都能正常显示了，但是视频仍然不能显示，我就觉得很奇怪，最后我发现是因为视频的大小太大了，GitHub加载不了，然后我就通过cloud convert转换了视频的清晰度等，最终顺利生成了链接

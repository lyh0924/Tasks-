# Git学习心得  
***
在接触 Git 之前，我对于它是一无所知的，不知道有什么很好的平台去储存我的笔记或者代码，更不知道什么是GitHub，什么叫做代码托管中心。直到真正上手 Git，我才发现原来有这样一个专业又便捷的工具，把我之前乱糟糟的方式理顺了不少。
 
一开始学命令的时候，是非常懵的， add   commit   push   pull  这些指令记了又忘，有时输错命令就会很不知所措，不知道怎么回退，怎么解决。但通过这些天对于Git和GitHub的接触，才发现Git 的核心逻辑特别简单——就是记录每一次文件的修改，想回到哪个版本、想对比哪次改动，都能随时查，再也不用靠记忆存文件版本了。
 
这几天又新了解到了Git在小组协作写代码工作的作用，觉得Git功能很强大。我了解到多人一起改同一个项目，可能会出现“我改了这个文件，他也改了，最后大家的版本都覆盖了”的问题，大大影响了进度。而用Git之后，每个人在自己的分支上改代码，提交后再合并到主分支，就算改了同一行也能清晰看到冲突提示，解决冲突的过程也相对轻松。
 
当然，Git也不是完全没门槛，特别容易上手。比如远程仓库的连接、PR的提交流程，一开始操作的时候总容易出错，这都需要耐心和时间一步步去理解和掌握。
 
总的来说，Git 不是那种“学完就忘”的工具，越实操越能提高掌握度，后续我也想再深入学学分支管理的高级用法，能让自己的代码与他人的协作更高效。

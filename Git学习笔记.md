# Git学习笔记  
***  
## 使用Git Bash提交hello.md文档到远程仓库的完整步骤  
1. 首先要有一个.md格式的文档（此处为hello.md）  
2. 我把我所创建的的.md的文档全部放在了桌面上的test文件夹中  
3. 打开Git Bash后进入test文件夹  
   *  cd~/Desktop/test  
4. 初始化本地git仓库：git init   
4. 添加新笔记到Git追踪  
   * git add hello.md(如果是全部.md则可写成git add *.md)    
5. 提交到本地仓库  
   * git commit -m "hello.md"    
6. 关联远程GitHub仓库：git remote add origin https://github.com/lyh0924/Tasks.git 
6. 第一次推送到远程仓库  
   * git push -u origin main  
7. 后续再推送：  
    * git add .
    * git commit -m "更新说明"  
    * git push   
***  
## Git工作流程  
1. **新创建一个文件**    
    1. 在本地仓库下通过Git Bash新建一个文件，可输入**touch 文件名**，即可生成，此时的文件处于**工作区**  
    2. 后通过**git add .(或git add 文件名)**，添加文件到**暂存区**  
    3. 最后再通过**git commit -m "add 文件名"**，将文件提交到**仓库**  
2. **修改已有文件**   
    1. 通过vi 文件名修改已建好的文件  
    2. 之后步骤跟新建一个文件中的第二第三步一样   
***  
## Git常用指令  
1. **查看修改的状态**：git status  
2. **添加工作区到暂存区**：git add .  
3. **提交暂存区到本地仓库**:git commit -m"注释内容"  
4. **查看提交日志**：git log  
5. **版本回退（版本切换）**：git reset --hard commitID(commitID可以通过git log查看)  
6. **查看已经删除的记录**：git reflog  
***  
## 分支  
1. **查看本地分支**：git branch  
2. **创建本地分支**：git branch 分支名  
3. **切换分支**：git checkout 分支名  
4. **切换的同时创建一个新的分支**:git checkout -b 分支名   
5. **合并分支**：git merge 分支名  
6. **删除分支**：git branch -d 分支名（-D强制删除）   
7. **解决冲突（不同的人在同一文件的同一处进行了不同修改）**：
    1. 处理文件的冲突地方  
        1. 输入vim "文件名.md"  
        2. 找到冲突区域标志（<<<<<<<和>>>>>>>和=======）  
        3. 按i进入插入模式，可进行编辑 
        4. 编辑完成后按ESC键退出插入模式，输入:wq保存并退出 
    2. 将处理完的文件加入暂存区  
    3. 最后提交到仓库  
***  
## clone  
1. **目的**：完整复制一个已经存在的远程Git仓库到本地的命令  
2. **核心作用**：  
    * 把Github等远程仓库的所有文件，提交历史，分支和标签都下载到本地，可以进行离线开发，查看历史，切换分支  
    * 可以进行**多人协作开发**，团队成员通过clone项目获取代码，之后各自在本地修改，再通过push或pull同步到远程仓库   
    * 保留Git功能，可直接在本地执行所有Git操作，和原仓库完全一致  
3. **操作**：  
    1. 复制远程仓库地址  
    2. 执行命令：git clone 地址  
***  
## 抓取和拉取  
1. **场景**：远程分支和本地分支一样，都可以进行merge操作，只是要先把远端仓库里的更新都下载到本地，再进行操作  
2. **抓取命令**：git **fetch**[remote name][branch name]  
    * 目的是将仓库里的更新都抓取到本地，不会进行合并，如果不指定远端名称和分支名，则抓取所有分支  
3. **拉取命令**：git **pull**[remote name][branch name]  
    * 将远端仓库的修改拉到本地并自动进行合并，等同于**fetch+merge** ，如果不指定远端名称和分支名，则抓取所有并更新当前分支   
***  
## 使用Git Bash提交是否麻烦以及更简单方法  
1. 我认为刚开始用bash提交时，可能是由于我个人原因，与正常的提交流程有一些偏差，弄了SSH密钥等步骤之后才能正常提交文档到远程仓库，所以有点麻烦，但经过第一次的准备工作后，后续再提交文档等，只需要输入简短的三行指令，于我而言是不算麻烦的。  
2. 更便捷的方法(**适合现阶段**)  
    1. **VS Code内置的Git工具**  
        1. 打开项目文件夹，左侧点击【源代码管理】图标（crtl+Shift+G）  
        2. 在【更改】区域，点击文件旁的+号，把修改加入暂存区  
        3. 在顶部输入框写提交信息，点击对钩号（提交按钮）  
        4. 提交后，点击【...】进行推送到远程仓库  
    2. **GitHub Desktop（GitHub 官方可视化客户端）**  
        1. 下载安装 GitHub Desktop，登录GitHub 账号   
        2. 克隆/添加你的本地仓库  
        3. 在界面里勾选要提交的文件，填写提交说明，点击「Commit to main」
        4. 点击「Push origin」推送到远程

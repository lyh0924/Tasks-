# GitHub学习笔记
## 1. 关于 Geek 与 Geek 精神
 
### 什么是 Geek？
 
Geek 原指“**极客**”，最早用来形容痴迷于计算机、技术的小众群体。如今它的含义更广泛：*对某一领域（技术、游戏、艺术等）抱有极致热爱与好奇心，愿意花大量时间钻研、动手实践，并乐于分享的人*。
 
### Geek 精神是什么？
 
- **好奇心驱动**：永远对“为什么”和“怎么做”保持敏感
​
- **动手实践**：不只是看，更要亲手做、试错、迭代
​
- **开源与分享**：相信知识应该流动，乐于把自己的成果和经验分享给社区
​
- **追求极致**：对细节、效率、美感有执念，总想把事情做到更好
​
- **拒绝平庸**：喜欢用技术/创意解决问题，而不是随波逐流
 
 
 
## 2. GitHub 与核心概念理解
 
### GitHub 是干什么的？
 
GitHub 是一个基于 Git 的*代码托管与协作平台*，核心作用：
 
- 存放代码、文档、配置等文件，用 Git 管理版本历史
​
- 让**多人**协作开发同一个项目（比如开源软件、课程作业）
​
- 成为**开发者**的“技术简历”，展示你的项目、学习轨迹和技术能力
 
### Pull Request（PR）
 
- 我的理解：你在自己的分支/仓库里改完代码后，向原仓库发起的「**合并请求**」，告诉项目维护者：“我做了这些修改，要不要合并到主分支？”
​
- 学习来源：GitHub 官方文档、B站 Git 教程、开源项目贡献指南
 
### Fork
 
- 我的理解：把别人的仓库“**复制一份**”到自己的 GitHub 账号下，你可以在自己的 Fork 仓库里任意修改，不会影响原仓库
​
- 学习来源：百度搜索
 
 
 
## 3. 关于“打开 GitHub 花了半天”的经历
 
* 我没有遇到过完全打不开 GitHub 的情况，但是我在刚下载的时候，想要打开bash通过输入代码来检查是否正确下载，因为在网上看的教程里面要输入某个代码之后，能看到下载的版本号才算正确安装了。  
* 但我其实一开始是应该是正确下载了的，也能正常使用，但是由于一开始对这些软件书写的不熟悉。导致我在输入代码的时候，在字母和减号中间没有输入空格，理所应当的认为不需要输入空格，也没注意要输入空格，所以就导致一直错误，花了很多时间研究，后面才发现是这么一个小细节，所以在今后的学习中要多注意细节方面问题。
 
 
 
## 4. 这份文档的整洁规范说明
 
这份文档遵循了以下 Markdown 规范，因此称得上整洁规范：
 
*  层级清晰：用  # 、 ## 、 ###  区分标题层级，结构一目了然
​
* 列表规范：用○做无序列表，用  1.  做有序列表，缩进对齐
​
* 重点突出：用 加粗 标记核心概念，用  代码块  标记专业术语
​
* 分隔线：用  ---  分隔不同模块，视觉上更清爽
​
* 可读性优先：每行文字不超过屏幕宽度，避免大段无分割的文字，方便阅读和维护
